# API

### Application Programming Interface

컴퓨터나 컴퓨터 프로그램 사이의 연결
컴퓨터 사이의 중계 계층

<br><br>

## API 의미

* 라이브러리 및 프레임워크를 설명하는 명세서
* Web API (✅ 일반적으로 말하는 API)
* Web Socket API

<br><br>

## API 종류

* `private` : 내부적으로 사용. 해시키를 하드코딩 해두고 이를 기반으로 서버와 서버 간 통신
* `public` : 모든 사람이 사용 가능. 많은 트래픽을 방지하기 위해 하루 요청수 제한, 계정당 개수 제한 등으로 관리

<br><br>

## API 장점

1. 제공자는 서비스의 중요한 부분을 드러내지 않아도 됨 (DB 설계 구조, 정보 등)
2. 사용자는 해당 서비스가 어떻게 구현되는지 몰라도 되며, 필요한 정보만 받을 수 있음
3. open API의 경우, 앱 개발 프로세스를 단순화 시키고 시간과 비용을 절약
4. 제공자의 경우 API를 만들면 내부 프로세스가 수정되었을 때 매번 API를 수정하지 않아도 되며, 내부가 변경되어도 사용자에게 영향 주지 않고도 변경이 가능
5. 제공자는 데이터를 한 곳에 모을 수 있음
6. 제공자는 제3자가 만든 앱을 통해 데이터를 수집하거나, 해당 서비스를 확장할 수 있으며 홍보에도 도움됨
